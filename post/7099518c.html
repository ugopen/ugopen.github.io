NX二次开发-使用NXOpen::Body::GetFaces()接口导致软件崩溃问题

发表于 2025-08-14 分类于 疑难杂症

编译环境

NX 11.0.2.7 MP13
Visual Studio 2017（v141）
注：原则上 NX 11.0.2.7 MP13 应该使用Visual Studio 2013（V120）编译

问题描述

使用NXOpen::Body::GetFaces()接口获取实体所有面，当面的数量达到512个，就会导致NX软件崩溃。

问题解决

以上问题可能是没有使用对应的Visual Studio 2013（V120）编译所致，临时解决可以使用以下UFUN接口解决，不过建议使用NX对应的VS版本编译。
