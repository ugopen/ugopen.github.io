NX二次开发-UFUN对象（tag_t）转换成NXOPEN对象（NXObjectManager）

发表于 2020-09-14 分类于 NX二次开发

说明

通用NX对象：

按照是否被保存到部件文件中，分为永久和临时NX对象

1.永久NX对象：全部继承TaggedObject

2.临时NX对象：用来辅助创建永久对象，或者临时在NX用户界面中显示的对象。

通用NX对象的操作

通用对象标识：Tag Handle TaggedObject typedef unsigned int tag_t 所有NX对象都是通过Tag被标识和访问。Tag在代码中用tag_t表示 用于标识对象的变量tag_t 只在当前Session中固定不变。NULL_TAG是保留的特殊Tag,表示暂时不标识任何对象的Tag。

Handle 是NX对象句柄，每个NX对象都有个字符串编码的Handle,他在文件中是唯一的，是可以随文件一起保存，永远不变。

可以直接调用UF_TAG_ask_handle_of_tag和UF_TAG_ask_tag_of_handle实现Tag与Handle的互换。

TaggedObject 是NXopen中所有NX对象的基类。

TaggedObject ->Tag 得到NX对象的Tag标识；

已知Tag对象标识时，可以通过NXObjectManager::Get(tag_t)得到TaggedObject 对象指针，再通过C++类型转换符dynamic_cast将TaggedObjeect对象指针转换为实际NX对象指针。
